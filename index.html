ShadowLogin

ShadowLogin is an academic prototype demonstrating how temporary, privacy-focused virtual accounts can help protect personal data during testing, onboarding, or research tasks. The system generates short-lived identities that reduce linkability between services, lower exposure to tracking, and provide a safe environment for UI testing and demonstrations.

This demo emphasizes privacy, design transparency, and educational value. It intentionally avoids connecting to any external services; all generated data is synthetic and stored locally for demonstration purposes only.

Generator

The generator produces disposable virtual accounts across multiple types. Select an account type below, tune the configuration, and generate a temporary profile. Generated accounts include optional fields (username, full name, email, phone, and avatar). Choose a country for phone formatting, set a privacy level, and select the account lifetime. These choices alter the privacy score ‚Äî a demo metric indicating relative identifiability.

About Us

ShadowLogin was created as an academic prototype by students and researchers focused on privacy-preserving demonstrations. The primary goal is to showcase how temporary virtual accounts can be instrumented to support UX testing and privacy education without exposing real user data.

Instructors and reviewers should note this is intentionally isolated from real authentication systems; it illustrates design patterns not production deployment.

What We Do

ShadowLogin demonstrates several features relevant to privacy-first prototypes:

  • Generate temporary accounts with selectable fields.
  • Adjust privacy level and observe a demo privacy score.
  • Track lifecycle with dashboard statistics and activity logs.
  • Share or deactivate demo accounts as part of testing workflows.
  • Collect feedback and ratings locally for evaluation.

What's New

Recent updates in this Advanced Edition include multi-type account generation, an improved dashboard, mobile responsiveness, demo chat, and enhanced privacy scoring. These changes deepen the educational value and make the demo suitable for presentation environments.

  • Multi-type account generation (Social, Email, Website, Gaming, Anonymous, Custom)
  • Persistent local storage and lifecycle management
  • Privacy score and visual indicators
  • Share & deactivate actions
  • Responsive, mobile-first UI with sticky navigation

UI & General Requirements

This demo follows a mobile-first, dark cybersecurity theme. Navigation is sticky and uses smooth scroll with active section highlighting. Animations are subtle and designed to draw attention to lifecycle events (account generation and expiration) without being distracting. The prototype purposefully avoids external integrations ‚Äî it is an isolated academic demonstration.
